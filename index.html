What we should expect from tests?

What we should expect from tests?

Confidence in what we code, both because it works and because it does not break previously working code

What we should expect from tests?

We cannot totally rely on manual testing because

  • We will miss the whole picture
  • It is time expensive
  • It does not scale
  • It includes a fallible component

What we should expect from tests?

So we want non manual tests

  • Automated
  • Deterministic
  • Fast to execute

What we should expect from tests?

And we want this

to have fast feedback loops

so we detect failures as soon as possible, when they are cheap

Itâ€™s all about the feedback!

What we should expect from tests?

Automated

We must be able to run all our tests with the minimum interaction possible

And we can run them without having to prepare all the test setup

So they can be introduced in CI pipelines

What we should expect from tests?

Deterministic

Always the same result with the same input

They will have to deal with variable factors (ie: time, randomness)

What we should expect from tests?

Fast

  • Slow tests provide slow feedback cycles
  • If they take ages to run, we will simply not run them
  • We trend to disconnect if something takes too long, losing focus

How to write better tests

Stick to the structure

AAA: Arrange, Act, Assert

                
public function testCartIsEmptyAfterCheckout(): void {
    //Arrange
    $this->cart->addItem(2, new Tuneladora());
    //Act
    $this->cart->checkout();
    //Assert
    $this->assertTrue($this->cart->isEmpty());
}

How to write better tests

Stick to the structure

GWT: Given, When, Then

                
public function testCartIsEmptyAfterCheckout(): void {
    $this->givenIHaveACartWithItemsOnIt();
    $this->whenTheCartIsCheckedOut();
    $this->thenTheCartIsEmpty();
}

How to write better tests

Test behavior, not implementation

Tests should tell you if your application works, not how it is coded

Usually there are many ways of solving a problem. Our tests should not fail if we exchange between valid ways of doing it

How to write better tests

Use builders for data fixtures

                
public function testPromoteUserToAdmin {
    $user = UserBuilder::regularUser();
    $user->promoteUserToAdmin();
    $this->assertTrue($user->isAdmin());
}

Now a change in User constructor implies only updating the builder, not all the tests doing a new User<

How to write better tests

Provide factory methods for commonly used configurations

Provide a way of customizing each parameter for concrete, exceptional configurations

                
class UserBuilder {
    //This sets default/random values for all not provided parameters
    public static function withParameter(array $parameters): User {}

    //Concrete methods internally using withParameters()
    public static function regularUser(): User {}
    public static function adminUser(): User {}
}
